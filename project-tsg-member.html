台鋼會員系統

Web 會員系統

從註冊到兌換的完整會員旅程設計 — 以分步驟引導降低放棄率，用統一設計規範確保每個頁面都說同一種語言。

角色：產品設計師 時間：2024.1 – 2024.6 類型：桌面 + 手機響應式

基於保密協議，本頁僅展示部分畫面與流程，不包含完整介面內容。

設計原因

設計元素選擇原因
主色調金黃色（暖金色系）呼應台鋼品牌識別，傳達「尊榮會員」價值感
輔助色深灰/黑色 + 白色背景高對比確保可讀性，金+黑傳達品牌質感
字級系統H1: 44px → Body Small: 12px（10 級）所有行高統一 1.5x，確保全站一致性
註冊方式分步驟 Popup用戶始終看到主頁背景，降低離開感
CTA 按鈕金黃色填滿按鈕在白色背景上高度醒目，引導完成操作

專案亮點

  • 從 0 建立設計語言 — Typography 十級字階 + Layout Grid，確保跨頁面視覺一致性
  • 每一種例外都被考慮到 — 空白、填寫中、錯誤、成功，四種狀態完整覆蓋真實場景
  • 先見森林，再見樹木 — Sitemap 清楚劃分功能層級，從全局架構推導每一頁的存在理由
  • 讓點數規則不再是黑箱 — 視覺化流程圖拆解獲取、消耗、到期的完整生命週期
  • 每一步只做一件事 — 5 步 Popup 漸進式註冊，用戶始終看到主頁背景，安全感不中斷

解決的痛點

痛點設計解法
註冊流程冗長，中途放棄率高5 步驟 Popup 流程，每步聚焦一件事
忘記密碼找回困難多種驗證方式（OTP / Email），流程視覺化引導
點數機制不透明點數流程圖清楚呈現獲取、兌換、到期規則
優惠券管理混亂分類呈現（可使用/已使用/已過期）+ 到期提醒
個人資料修改不便即時編輯 + 表單驗證與錯誤狀態提示
會員系統缺乏一致性建立完整設計規範（Typography + Layout）

使用者流程

註冊流程（5 步驟 Popup）

點「馬上開始」→ 輸入手機號碼 → OTP 驗證碼 → 設定密碼 → 填寫個人資訊 → 註冊成功

忘記密碼流程

忘記密碼 → 輸入手機號碼 → OTP 驗證 → 重設密碼 → 完成

點數兌換流程

查看堡幣餘額 → 堡幣紀錄 → 點數商城 → 兌換確認
