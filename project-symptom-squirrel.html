松鼠醫生App與後台

醫療 App

讓不會打字的阿公阿嬤，也能輕鬆記錄每天的健康狀態。一套串聯病患 App 與醫護後台的雙端系統，用最低的操作門檻，換來最完整的照護數據。

角色：UIUX 設計師 時間：2025.7 – 2026.1 類型：iOS App + Web 後台

基於保密協議，本頁僅展示部分畫面與流程，不包含完整介面內容。

設計原因

設計元素選擇原因
主色調墨綠色（Teal Green）傳達醫療專業感與安心感，綠色系具有「健康、自然、平靜」的心理暗示
輔助色綠/黃/紅三色系統交通燈隱喻，高齡用戶無需閱讀文字即可直覺理解
字體風格SF Pro + 大粗體遵循 iOS HIG，問卷頁字級極大，專為視力退化的高齡用戶設計
圖示風格松鼠吉祥物 + 症狀插圖降低醫療嚴肅感，情感化設計增加使用動機
互動風格正向回饋動畫每次完成任務給予即時獎勵感，搭配 Streak 強化習慣養成
登入方式QR Code 取代帳密長輩對鍵盤輸入有障礙，掃碼零輸入門檻

專案亮點

  • 病患記錄，醫師秒懂 — App 端日常數據即時同步後台，AI 自動產出回診摘要，讓每次看診都有據可依
  • 為「最不會用手機的人」而設計 — 超大字體、圖文點選、掃碼免密碼，每個細節都在降低一道門檻
  • 讓記錄變成一種期待 — 連續打卡養成植物、松鼠陪伴鼓勵，四重正向回饋讓健康習慣自然留存
  • 一眼看懂的健康狀態 — 紅黃綠交通燈直覺映射血壓數值，不需讀數字也能掌握趨勢
  • 從零到一的系統思考 — 完整 User Flow 涵蓋 5 條核心路徑，展現端到端的設計規劃力

解決的痛點

對象痛點設計解法
高齡病患視力退化，看不清小字問卷頁面採用超大粗體中文字，每題一頁，減少視覺干擾
病患回診前難以描述症狀Symptom Diary 提供 30+ 症狀圖文選單，每日點選記錄

病患健康記錄習慣難以持續Streak 連續打卡 + 植物生長視覺化

醫師回診前缺乏病患日常數據後台查看追蹤紀錄，AI 自動彙整報告

使用者流程

流程 A — 登入與醫師連動

開啟 App → 顯示 QR Code → 醫師用後台掃描 → 帳號自動綁定 → 進入主頁

流程 B — 每日問卷填寫

主頁提示 → Start Assessment → 超大字體逐題填寫 → Submit → 慶祝動畫 → 引導進入症狀紀錄

介面流程 C — 血壓記錄

Tracking → BP Diary → Log Blood Pressure → 輸入 SYS/DIA/PUL → Save → 查看月曆（綠/黃/紅色點）

介面流程 D — 症狀紀錄

Tracking → Symptoms → 分頁症狀清單（30+ 症狀）→ Follow-up 追問 → Add to Diary → Streak + 植物生長動畫

介面流程 E — 飲食記錄

Tracking → Diet → 拍照或文字記錄 → 輸入餐點名稱 → Gallery 查看歷史
