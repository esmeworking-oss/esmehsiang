Design Process

1

發想與探索 Ideation

Antigravity
  • 透過 AI 快速產出多種設計方向與視覺概念
  • 在短時間內探索大量可能性，避免過早收斂
  • 將模糊的想法轉化為具體的視覺參考
效率提升：原本需要數小時的 moodboard 蒐集，縮短至 30 分鐘內完成
2

設計與迭代 Design & Iteration

Figma Make
  • 利用 AI 輔助快速生成 UI 元件與版面配置
  • 根據設計規範自動產出一致的元件變體
  • 加速從線框稿到高保真設計稿的轉換
效率提升：重複性的元件設計工作減少約 40%，更多時間投入在體驗細節
3

原型與開發 Prototyping & Development

Claude Code
  • 以 Vibe Coding 方式與 AI 協作，快速將設計稿轉為可互動的原型
  • 即時產出前端程式碼，不需等待工程師排程
  • 設計師主導開發節奏，邊做邊調整
效率提升：原型製作速度提升 3-5 倍，設計驗證週期從週縮短至天
4

落地與上線 Launch & Deploy

Framer
  • No-code 快速建置響應式網站
  • 從設計到上線一人完成，無需額外開發資源
  • 支援即時更新，隨時根據回饋調整內容
效率提升：網站上線時間從數週壓縮至數天

Continuous Iteration

發想 → 設計 → 原型 → 上線 → 回饋 → 優化 ↻ 持續循環 — 每次專案都在優化流程
  • 持續更新 — 隨著 AI 工具快速演進，即時導入更好的解決方案
  • 持續迭代 — 每次專案結束後回顧流程，找出可以再加速的環節
  • 保持彈性 — 不同專案根據需求靈活調整工具組合與流程順序

Case Study: 這份作品集網站

從零打造一個產品設計師求職作品集網站，包含 Loading 動畫、響應式設計、5 個專案詳細頁、研究展示區塊——全程以 AI 工具加速，一人完成設計到上線。

1
發想與視覺探索
Antigravity
  • 輸入關鍵字「簡潔、留白、讓作品說話」，AI 快速生成多組視覺風格方向
  • 從數十種配色與版面構想中，篩選出「清新藍白 + 大量留白」的設計基調
  • 參考 umao.co.jp 的設計語言，透過 AI 延伸出適合個人品牌的視覺方案
視覺探索從 2-3 天壓縮至半天
2
UI 設計與元件製作
Figma Make
  • 利用 AI 輔助生成專案卡片（Project Card）的多種排版變體
  • 快速產出 Hero Section、About、Contact 等區塊的 UI 元件
  • AI 自動生成一致的間距與字級系統，確保全站視覺統一
5 個專案頁面的 UI 元件從 3 天縮短至 1 天
3
Vibe Coding 開發
Claude Code
  • 網站架構 — 透過對話描述需求，AI 直接產出 HTML/CSS/JS 完整程式碼
  • Loading 動畫 — 描述「首次造訪顯示 Loading，重複造訪跳過」，AI 實作 sessionStorage 記憶邏輯
  • 滾動動畫 — 描述「區塊滾入時淡入上移」，AI 用 IntersectionObserver 完成
  • 專案詳細頁 — 提供每個專案的痛點、流程、設計原因等內容，AI 自動套版生成 5 個獨立頁面
  • 響應式設計 — 一句「確保 Desktop / Tablet / Mobile 都能正常顯示」，AI 完成所有斷點調整
  • 即時迭代 — 看到不滿意的地方，直接用自然語言描述修改，秒級回應
整站開發（含 5 個專案頁）從預估 2-3 週壓縮至 3 天
4
上線與持續更新
Framer / GitHub Pages
  • 快速部署上線，即時預覽效果
  • 收到回饋後，搭配 Claude Code 即時修改並重新部署
  • 持續新增內容（如這份 AI 流程頁面本身），保持作品集的鮮活度
部署 + 修改循環從「每次半天」縮短至「每次 30 分鐘」

My Perspective

AI 不會取代設計師，但善用 AI 的設計師會取代不用 AI 的設計師。

我把 AI 當作設計流程中的加速器——它處理重複性工作，我專注在策略思考、使用者洞察、和體驗細節。這讓我能以更少的時間交付更多的價值，同時維持設計品質。
